# P2.5 Frontend Foundation Implementation Plan

## 1. Overview and Objectives

### 1.1 Purpose
This document provides detailed step-by-step instructions for implementing the frontend foundation for The Strengths Toolbox website, including Blade layouts, components, Tailwind CSS configuration, and Alpine.js setup.

### 1.2 Scope
This implementation plan covers tasks P2.5.1 through P2.5.8:
- **P2.5.1**: Create main layout (app.blade.php)
- **P2.5.2**: Create admin layout
- **P2.5.3**: Configure Tailwind CSS theme
- **P2.5.4**: Create base CSS components
- **P2.5.5**: Set up Alpine.js components
- **P2.5.6**: Create header component
- **P2.5.7**: Create footer component
- **P2.5.8**: Create navigation component

### 1.3 Success Criteria
- Main and admin layouts created
- Tailwind CSS properly configured
- Base CSS components defined
- Alpine.js integrated and working
- Header, footer, and navigation components created
- Responsive design implemented
- All components are reusable and maintainable

## 2. Prerequisites

### 2.1 Required Knowledge
- Laravel Blade templating
- Tailwind CSS
- Alpine.js
- HTML/CSS/JavaScript
- Responsive design principles

### 2.2 Dependencies
- Task P1.1.5 completed (Vite and build tools set up)
- Task P1.1.6 completed (Tailwind CSS and Alpine.js installed)
- Task P1.3.5 completed (Blade layout structure created)
- Task P2.1.4 completed (SEOService for meta tags)

### 2.3 Reference Documents
- Frontend Architecture: `documentation/01-architecture/04-frontend-architecture.md`
- Laravel MVC Architecture: `documentation/01-architecture/02-laravel-mvc-architecture.md`

## 3. Frontend Architecture Overview

### 3.1 Component Structure
```
resources/views/
├── layouts/
│   ├── app.blade.php          # Main web layout
│   └── admin.blade.php        # Admin layout
├── components/
│   ├── header.blade.php        # Site header
│   ├── footer.blade.php       # Site footer
│   └── navigation.blade.php   # Navigation menu

```

## 4. Task P2.5.1: Create Main Layout (app.blade.php)

### 4.1 Overview
Create the main Blade layout that will be used for all public-facing pages, including proper meta tags, asset loading, and component structure.

### 4.2 Step-by-Step Implementation

**File: `resources/views/layouts/app.blade.php`**
```blade
<!DOCTYPE html>
<html lang="{{ str_replace('_', '-', app()->getLocale()) }}">

    <meta charset="utf-8">

    <meta name="csrf-token" content="{{ csrf_token() }}">

    {{-- SEO Meta Tags --}}
    @include('partials.meta')

```

## 8. Task P2.5.5: Set up Alpine.js Components

**File: `resources/js/bootstrap.js`**
```javascript
import axios from 'axios';
window.axios = axios;

window.axios.defaults.headers.common['X-Requested-With'] = 'XMLHttpRequest';

// Set CSRF token for all requests
let token = document.head.querySelector('meta[name="csrf-token"]');
if (token) {
    window.axios.defaults.headers.common['X-CSRF-TOKEN'] = token.content;
}
```

### 8.3 Create Alpine.js Components

**File: `resources/js/components/mobileMenu.js`**
```javascript
export function mobileMenu() {
    return {
        open: false,
        toggle() {
            this.open = !this.open;
        },
        close() {
            this.open = false;
        }
    };
}
```

**File: `resources/js/components/dropdown.js`**
```javascript
export function dropdown() {
    return {
        open: false,
        toggle() {
            this.open = !this.open;
        },
        close() {
            this.open = false;
        }
    };
}
```

**File: `resources/js/components/modal.js`**
```javascript
export function modal() {
    return {
        show: false,
        open() {
            this.show = true;
            document.body.style.overflow = 'hidden';
        },
        close() {
            this.show = false;
            document.body.style.overflow = '';
        }
    };
}
```

Update `resources/js/app.js` to include components:
```javascript
import './bootstrap';
import Alpine from 'alpinejs';
import { mobileMenu } from './components/mobileMenu';
import { dropdown } from './components/dropdown';
import { modal } from './components/modal';

// Register Alpine data
Alpine.data('mobileMenu', mobileMenu);
Alpine.data('dropdown', dropdown);
Alpine.data('modal', modal);

window.Alpine = Alpine;
Alpine.start();
```

### 12.4 Common Issues and Solutions

#### Issue: Tailwind styles not applying
**Solution**: 
- Clear Vite cache: `npm run build`
- Restart Vite dev server: `npm run dev`
- Verify `tailwind.config.js` content paths include your Blade files
- Check browser cache

#### Issue: Alpine.js not working
**Solution**: 
- Check Alpine is imported in `app.js`
- Verify Alpine.start() is called
- Check browser console for errors
- Ensure x-data attributes are correct

#### Issue: Assets not loading
**Solution**: 
- Verify `@vite()` directive is in layout
- Check Vite dev server is running
- Clear Laravel cache: `php artisan cache:clear`
- Check `vite.config.js` configuration

#### Issue: Mobile menu not working
**Solution**: 
- Verify Alpine.js mobileMenu component is registered
- Check x-data="mobileMenu()" is on correct element
- Verify x-show and x-transition directives
- Check for JavaScript errors in console
